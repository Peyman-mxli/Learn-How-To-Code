Bienvenida

Sistema de Biblioteca

Aprenderemos los conceptos base de POO creando un sistema real: gestión de libros. Sin herencia ni clases abstractas aún — solo clases, atributos, métodos y objetos.

📦

Clase

Plantilla para crear objetos. Define qué datos tendrá y qué puede hacer.

🔵

Objeto

Instancia concreta de una clase. Cada libro es un objeto diferente.

📋

Atributo

Dato que describe al objeto: título, autor, año, disponibilidad.

⚙️

Método

Acción que puede hacer el objeto: prestar, devolver, mostrar info.

¿Qué vamos a construir?

Un sistema con dos clases: Libro (representa un libro individual) y Biblioteca (gestiona una colección de libros). Cada parte se construye sobre la anterior.

💡 Consejo antes de empezar Lee cada paso completo antes de escribir código. Entender qué hace el método antes de cómo escribirlo facilita mucho el proceso.

La clase Libro

Primero definimos la estructura: qué datos tiene un libro y cómo se inicializa. Aún sin métodos de acción.

Atributos que necesitamos

AtributoTipoDescripción

Constructor __init__()

El constructor se ejecuta automáticamente cada vez que creamos un objeto con Libro(...). Recibe los datos iniciales y los guarda en el objeto usando self.

libro.py — estructura inicial 
class Libro:
    """Representa un libro en la biblioteca."""

    def __init__(self, titulo: str, autor: str, año_publicacion: int):
        """
        Constructor: se ejecuta al crear un objeto Libro.
        Parámetros: titulo, autor, año_publicacion
        """
        self.titulo          = titulo
        self.autor           = autor
        self.año_publicacion = año_publicacion
        self.disponible      = True   # siempre empieza disponible


# ── Uso ───────────────────────────────────────────────────
libro1 = Libro("Cien años de soledad", "Gabriel García Márquez", 1967)
libro2 = Libro("Don Quijote", "Miguel de Cervantes", 1605)

# Acceder a atributos directamente
print(libro1.titulo)       # Cien años de soledad
print(libro1.disponible)   # True

¿Qué es self?

self es una referencia al objeto actual. Cuando escribes self.titulo = titulo, estás guardando el dato dentro de ese objeto específico. Cada objeto tiene su propia copia de los atributos.

❌ Sin self — dato perdido
def __init__(self, titulo):
    titulo = titulo  # variable local
    # el dato desaparece aquí
✅ Con self — dato guardado
def __init__(self, titulo):
    self.titulo = titulo  # guardado en el objeto
    # accesible desde cualquier método
📝 Nota sobre tipos Los : str, : int en los parámetros son anotaciones de tipo (type hints). Son opcionales y no afectan el funcionamiento, pero hacen el código más legible.

Parte Base

Los 3 métodos de acción

Ahora agregamos prestar(), devolver() y mostrar_info(). Cada uno modifica o consulta el estado del objeto.

libro.py — clase completa (parte base) 
class Libro:
    """Representa un libro en la biblioteca."""

    def __init__(self, titulo: str, autor: str, año_publicacion: int):
        self.titulo          = titulo
        self.autor           = autor
        self.año_publicacion = año_publicacion
        self.disponible      = True

    # ── 1. prestar() ─────────────────────────────────────
    def prestar(self) -> None:
        """Presta el libro si está disponible."""
        if self.disponible:
            self.disponible = False
            print(f'✓ El libro "{self.titulo}" ha sido prestado.')
        else:
            print(f'✗ El libro "{self.titulo}" ya está prestado.')

    # ── 2. devolver() ────────────────────────────────────
    def devolver(self) -> None:
        """Devuelve el libro a la biblioteca."""
        self.disponible = True
        print(f'✓ El libro "{self.titulo}" ha sido devuelto.')

    # ── 3. mostrar_info() ────────────────────────────────
    def mostrar_info(self) -> None:
        """Muestra la información completa del libro."""
        estado = "Disponible" if self.disponible else "Prestado"
        print(f"📖 Título : {self.titulo}")
        print(f"✍️  Autor  : {self.autor}")
        print(f"📅 Año    : {self.año_publicacion}")
        print(f"📌 Estado : {estado}")
        print()   # línea en blanco al final

Prueba el código

Ejemplo de uso 
libro1 = Libro("Cien años de soledad", "Gabriel García Márquez", 1967)

libro1.mostrar_info()   # muestra todo
libro1.prestar()        # presta
libro1.prestar()        # intenta prestar de nuevo → mensaje de error
libro1.devolver()       # devuelve

Salida esperada en consola

✍️  Autor  : Gabriel García Márquez
📅 Año    : 1967

✓ El libro "Cien años de soledad" ha sido devuelto.

Puntos clave

  • if self.disponible: — leemos el atributo para decidir qué hacer
  • self.disponible = False — modificamos el estado del objeto
  • Los f-strings f"..." permiten insertar variables dentro del texto con {variable}
  • -> None indica que el método no retorna ningún valor

Validaciones y método clásico

Mejoramos la clase Libro para rechazar datos inválidos y poder identificar libros clásicos.

🎯 Reto 1

¿Qué debes agregar?

  • En __init__(): validar que titulo y autor no estén vacíos
  • En __init__(): validar que año_publicacion esté entre 1000 y 2025
  • Si una validación falla: imprimir un mensaje de error claro
  • Nuevo método es_clasico() que retorna True si el libro tiene más de 50 años
libro.py — con validaciones y es_clasico() 
class Libro:

    def __init__(self, titulo: str, autor: str, año_publicacion: int):

        # ── Validación 1: título no vacío ─────────────────
        if not titulo or titulo.strip() == "":
            print("❌ Error: el título no puede estar vacío.")

        # ── Validación 2: autor no vacío ──────────────────
        if not autor or autor.strip() == "":
            print("❌ Error: el autor no puede estar vacío.")

        # ── Validación 3: año dentro del rango válido ─────
        if año_publicacion < 1000 or año_publicacion > 2025:
            print(f"❌ Error: el año {año_publicacion} no es válido "
                  f"(debe estar entre 1000 y 2025).")

        # Guardar los atributos de todos modos
        self.titulo          = titulo
        self.autor           = autor
        self.año_publicacion = año_publicacion
        self.disponible      = True

    # ── Método es_clasico() ───────────────────────────────
    def es_clasico(self) -> bool:
        """
        Retorna True si el libro tiene más de 50 años de antigüedad.
        Retorna False en caso contrario.
        """
        return (2025 - self.año_publicacion) > 50

    # ... (los demás métodos no cambian)

Prueba las validaciones

Casos de prueba 
# ✅ Caso válido
libro_ok = Libro("Don Quijote", "Cervantes", 1605)
print(libro_ok.es_clasico())   # True  (más de 50 años)

# ✅ Libro reciente — no clásico
libro_nuevo = Libro("Dune", "Frank Herbert", 1965)
print(libro_nuevo.es_clasico())  # True (1965 también tiene +50)

libro_2020 = Libro("El problema de los tres cuerpos", "Liu Cixin", 2008)
print(libro_2020.es_clasico())   # False (menos de 50 años)

# ❌ Año inválido
libro_mal = Libro("Sin título", "Alguien", 500)
# Imprime: ❌ Error: el año 500 no es válido

# ❌ Título vacío
libro_vacio = Libro("", "Autor", 2020)
# Imprime: ❌ Error: el título no puede estar vacío.
✅ Nota sobre strip() titulo.strip() elimina espacios al inicio y al final. Así un título de " " (solo espacios) también se detecta como vacío. Es buena práctica usarlo en validaciones de texto.
💡 Mejora avanzada opcional En lugar de solo imprimir el error, puedes lanzar una excepción con raise ValueError("El título no puede estar vacío"). Esto detiene la ejecución y obliga al usuario del código a manejar el error correctamente.

Reto 2

La clase Biblioteca

Una clase que contiene objetos de otra clase. La lista libros guarda objetos Libro — esto se llama composición.

🎯 Reto 2

¿Qué debe tener la clase Biblioteca?

  • Atributo nombre: nombre de la biblioteca
  • Atributo libros: lista vacía que almacenará objetos Libro
  • Método agregar_libro(libro): añade un libro a la lista
  • Método buscar_por_titulo(titulo): retorna el libro o None
  • Método listar_disponibles(): muestra solo los disponibles
  • Método contar_libros(): retorna el total de libros
biblioteca.py 
class Biblioteca:
    """Gestiona una colección de objetos Libro."""

    def __init__(self, nombre: str):
        self.nombre = nombre
        self.libros = []   # lista vacía al inicio

    # ── agregar_libro() ───────────────────────────────────
    def agregar_libro(self, libro) -> None:
        """Agrega un objeto Libro a la colección."""
        self.libros.append(libro)
        print(f'📚 "{libro.titulo}" agregado a {self.nombre}.')

    # ── buscar_por_titulo() ───────────────────────────────
    def buscar_por_titulo(self, titulo: str):
        """
        Busca un libro por su título.
        Retorna el objeto Libro si lo encuentra, o None si no existe.
        """
        for libro in self.libros:
            if libro.titulo.lower() == titulo.lower():
                return libro
        return None   # no se encontró

    # ── listar_disponibles() ─────────────────────────────
    def listar_disponibles(self) -> None:
        """Muestra todos los libros que están disponibles."""
        print(f"\n📖 Libros disponibles en {self.nombre}:")
        disponibles = [l for l in self.libros if l.disponible]

        if not disponibles:
            print("   (No hay libros disponibles en este momento)")
        else:
            for i, libro in enumerate(disponibles, 1):
                print(f"   {i}. {libro.titulo} — {libro.autor} ({libro.año_publicacion})")

    # ── contar_libros() ───────────────────────────────────
    def contar_libros(self) -> int:
        """Retorna el número total de libros en la colección."""
        return len(self.libros)

Uso completo del sistema

main.py — todo junto 
from libro      import Libro
from biblioteca import Biblioteca

# Crear libros
libro1 = Libro("Cien años de soledad", "Gabriel García Márquez", 1967)
libro2 = Libro("Don Quijote",          "Miguel de Cervantes",    1605)
libro3 = Libro("El Principito",        "Antoine de Saint-Exupéry", 1943)

# Crear biblioteca y agregar libros
bib = Biblioteca("Biblioteca Central")
bib.agregar_libro(libro1)
bib.agregar_libro(libro2)
bib.agregar_libro(libro3)

print(f"\nTotal de libros: {bib.contar_libros()}")   # 3

# Prestar un libro
libro1.prestar()

# Ver solo los disponibles (debe mostrar libro2 y libro3)
bib.listar_disponibles()

# Buscar un libro
encontrado = bib.buscar_por_titulo("Don Quijote")
if encontrado:
    print(f"\nEncontrado: {encontrado.titulo}")
    encontrado.mostrar_info()

# Buscar uno que no existe
no_existe = bib.buscar_por_titulo("Harry Potter")
print(no_existe)   # None
🔍 List comprehension La línea [l for l in self.libros if l.disponible] crea una nueva lista filtrando solo los disponibles. Es equivalente a un for con un if, pero más corta. Ambas formas son correctas.

Preguntas conceptuales

Para reflexionar

Estas preguntas ayudan a afianzar los conceptos. Haz clic en cada una para ver una pista de respuesta.

1
¿Qué es el encapsulamiento y cómo lo vemos en la clase Libro?
▶
Encapsulamiento significa que los datos y los métodos que los manipulan viven juntos dentro de la clase. En Libro, el atributo disponible no se modifica directamente desde fuera — se hace a través de prestar() y devolver(), que validan el estado antes de cambiar nada. Esto evita que el dato quede en un estado inconsistente.
2
¿Por qué usar libro.prestar() en vez de libro.disponible = False?
▶
libro.disponible = False no valida nada: puedes prestar un libro que ya estaba prestado sin ningún aviso. prestar() revisa primero el estado y muestra un mensaje apropiado. Además, si en el futuro agregas un registro de préstamos, lo haces solo dentro de prestar(), no en todos los lugares del código que asignan el atributo.
3
¿Qué ventaja tiene la clase Biblioteca frente a usar una lista suelta?
▶
Con una lista suelta tendrías que repetir el código de búsqueda, filtrado y conteo en cada parte del programa. La clase Biblioteca centraliza esa lógica: si cambia la forma de buscar, lo modificas en un solo lugar. También agrupa datos (nombre, libros) con las operaciones que los usan — eso es POO.
4
Si crearas LibroDigital heredando de Libro, ¿qué agregarías?
▶
Posibles atributos extra: formato ("PDF", "EPUB"), tamaño_mb, url_descarga, descargas_simultaneas (límite de cuántos lo pueden "tener" al mismo tiempo). Métodos extra: descargar(), obtener_formato(). Heredaría prestar() y devolver() de Libro, quizás sobreescribiendo prestar() para permitir múltiples préstamos simultáneos.
