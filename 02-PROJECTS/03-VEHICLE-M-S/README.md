# 🧵 Project — Vehicle Management System
=======================================

# ✨ Overview

=============================================================================

The **Vehicle Management System** is a Python-based application
designed to manage different types of vehicles using
Object-Oriented Programming (OOP).

This project demonstrates how to structure a real-world system
using classes, inheritance, abstraction, and polymorphism.

It simulates a simplified vehicle system where multiple vehicle
types share common behavior while maintaining their own logic.

=============================================================================

🎯 Why This Project Matters

In real-world software:
Systems often manage multiple related entities that share
common characteristics but behave differently.

Without proper structure:
Code becomes repetitive, difficult to maintain, and hard to scale.

This project solves these problems by applying OOP principles
to build a clean, modular, and extensible system.

=============================================================================

📦 Core Concepts Applied

This project demonstrates:

Classes and Objects
Inheritance (Base → Derived classes)
Abstraction (Abstract base class)
Polymorphism (Different implementations of methods)
Encapsulation (Structured data within classes)

=============================================================================

🚘 Vehicles Implemented

The system currently includes:

Automóvil
Motocicleta
Camión
Bicicleta

Each vehicle inherits from a base class `Vehiculo`
and implements its own specific behavior.

This structure allows easy expansion for future vehicle types.

⚙️ How It Works

The system follows this architecture:

A base abstract class defines common behavior

Derived classes implement specific logic

Objects are created from different vehicle types

Polymorphism allows unified interaction

This creates a flexible and extensible system design.
