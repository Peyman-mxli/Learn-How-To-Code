# 🧵 Integration Testing

## ✨ Overview

In this module, I learn about Integration Testing in C#.

Integration testing means testing how different parts of a program work together.

While unit testing checks one small piece of code, integration testing checks the connection between multiple pieces.

For example, I may test if:

- A service works with a repository
- A controller works with a database
- A login system works with user validation
- A file system works with saved data
- Multiple classes communicate correctly

Integration testing is important because real applications are built from many connected parts.

---

## 🎯 What I will learn in this module

By the end of this module, I will be able to:

- Understand what integration testing is
- Understand the difference between unit testing and integration testing
- Test multiple classes working together
- Understand connected systems
- Test service and repository logic
- Test file read/write behavior
- Understand realistic application workflows
- Learn basic integration testing best practices
