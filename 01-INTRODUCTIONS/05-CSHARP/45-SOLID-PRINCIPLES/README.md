# 🧵 SOLID Principles

## ✨ Overview

In this module, I learn about SOLID Principles in C#.

SOLID is a group of five important object-oriented programming principles.

These principles help me write code that is clean, organized, flexible, and easier to maintain.

SOLID stands for:

- Single Responsibility Principle
- Open/Closed Principle
- Liskov Substitution Principle
- Interface Segregation Principle
- Dependency Inversion Principle

These principles are important because professional applications become large over time.

If I do not organize my code correctly, it becomes hard to understand, hard to test, and hard to change.

---

## 🎯 What I will learn in this module

By the end of this module, I will be able to:

- Understand what SOLID means
- Understand why SOLID principles are important
- Apply the Single Responsibility Principle
- Apply the Open/Closed Principle
- Apply the Liskov Substitution Principle
- Apply the Interface Segregation Principle
- Apply the Dependency Inversion Principle
- Write cleaner object-oriented code
- Reduce code duplication
- Improve maintainability
- Prepare for professional software architecture
