# 🧵 Garbage Collection

## ✨ Overview

In this module, I learn about **Garbage Collection (GC)** in C#.

When I create objects in a C# application, memory is allocated automatically. As my program continues running, some objects become unnecessary and are no longer used.

Instead of manually freeing memory like in some programming languages, C# uses a system called **Garbage Collection** to automatically clean up unused objects.

Understanding Garbage Collection helps me write more efficient applications, avoid memory leaks, improve performance, and understand what happens behind the scenes when objects are created and destroyed.

Garbage Collection is one of the most important features of the .NET runtime because it manages memory automatically and helps developers focus on building applications rather than manually handling memory cleanup.
