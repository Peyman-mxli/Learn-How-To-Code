# 09 - Classes & Objects

## Introduction

In this module, I learn the foundation of Object-Oriented Programming (OOP) in C#.

Classes and objects are among the most important concepts in C# because almost every real-world application uses them.

A class acts as a blueprint, while an object is a real instance created from that blueprint.

For example:

* A Player can be a class.
* An Enemy can be a class.
* A Car can be a class.
* A Student can be a class.

Then I can create many objects from those classes.

Object-Oriented Programming helps organize code, improve reusability, and make large projects easier to manage.

---

# What I Learn In This Module

### What a Class Is

I learn how a class defines properties and behaviors.

### What an Object Is

I learn how objects are created from classes.

### Fields

I learn how classes store data using fields.

### Methods Inside Classes

I learn how classes can contain methods.

### Creating Objects

I learn how to instantiate objects using the `new` keyword.

### Accessing Fields

I learn how to read and modify object data.

### Accessing Methods

I learn how to call methods from objects.

### Multiple Objects

I learn how many objects can be created from the same class.

---

# Why Classes & Objects Matter

Classes and objects are the foundation of modern software development.

With classes and objects, I can:

* Organize code better
* Create reusable systems
* Build games
* Build business applications
* Create simulations
* Manage large projects
* Follow Object-Oriented Programming principles

Without classes, programs become difficult to manage as they grow larger.

---

# Real-World Example

Blueprint:

Car

Objects:

* Red Car
* Blue Car
* Black Car

All cars share the same structure but contain different data.

The class defines the blueprint.

The object is the real thing created from that blueprint.
