# 02 - Syntax

## Introduction

In this module, I learn the basic syntax of C#.

Syntax is the set of rules that tells me how to write code correctly so that the compiler can understand and execute it.

Just like every spoken language has grammar rules, C# has syntax rules. Learning these rules is one of the most important steps when starting to program because every application, game, website, or software project relies on correct syntax.

By understanding syntax, I learn how to:

* Write valid C# code
* Create statements and instructions
* Organize my programs correctly
* Avoid common compiler errors
* Read and understand code written by other developers

This module focuses on the fundamental building blocks of C# syntax that I will use throughout my programming journey.

---

# What I Learn In This Module

### Statements

I learn how instructions are written in C# and why most statements end with a semicolon (`;`).

### Code Blocks

I learn how curly braces (`{ }`) are used to group code together.

### Main Method

I learn about the entry point of a Console Application and where execution begins.

### Namespaces

I learn how namespaces help organize code and avoid naming conflicts.

### Using Directives

I learn how the `using` keyword allows access to classes and functionality from different namespaces.

### Comments

I learn how to write notes inside my code using:

* Single-line comments
* Multi-line comments
* XML documentation comments

### Case Sensitivity

I learn why C# treats uppercase and lowercase letters differently.

### Formatting

I learn how proper indentation and formatting make code easier to read and maintain.

### Common Syntax Errors

I learn how to recognize and fix some of the most common mistakes beginners make.

---

# Why Syntax Matters

Without proper syntax, programs cannot compile.

Even a small mistake such as:

* Missing a semicolon
* Missing a bracket
* Misspelling a keyword
* Incorrect capitalization

can prevent a program from running.

Understanding syntax helps me write cleaner code, debug errors faster, and build a strong foundation for more advanced C# topics.
