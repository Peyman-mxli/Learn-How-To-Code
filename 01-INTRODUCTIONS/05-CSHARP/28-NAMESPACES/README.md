# 🧵 Namespaces

## Introduction

In this module, I learn about **Namespaces** in C#.

Namespaces help me organize code and avoid name conflicts.

When a project becomes larger, it may contain many classes, interfaces, records, enums, and other files.

Namespaces allow me to group related code together in a clean and professional way.

## Why It Matters

Namespaces are important because real projects contain many files.

Without namespaces, code can become messy and difficult to manage.

Namespaces help me:

* Organize classes
* Separate features
* Avoid duplicate name conflicts
* Improve readability
* Build professional project structures
* Work better in large applications

Namespaces are used in almost every C# project.

---

## Simple Example

```csharp
namespace MyApplication
{
    public class Student
    {
        public string Name { get; set; }
    }
}
```

This places the `Student` class inside the `MyApplication` namespace.

---

## Real-World Example

A school system may organize code like this:

```csharp
namespace SchoolSystem.Students
{
    public class Student
    {
        public string Name { get; set; }
    }
}
```

Another part of the project may use:

```csharp
namespace SchoolSystem.Teachers
{
    public class Teacher
    {
        public string Name { get; set; }
    }
}
```

This keeps the project organized by feature.
