🧵 Module —  Exception Handling — The except Statement
------------------------------------------------------------------------
The `except` statement is used to **handle errors** that occur inside a `try`
block. While `try` identifies risky code, `except` defines **how the program
should respond** when something goes wrong.

This module focuses on understanding:
- How `except` works
- Why specific exceptions matter
- How to structure clean and professional error handling

## ⚠️ Why except Is Important
Without `except`:
- Errors crash the program
- Users see confusing tracebacks
- Software becomes unreliable

With `except`:
- Errors are handled gracefully
- Programs can recover or fail safely
- User experience improves significantly
---------------------------------------------------------------------
## 🧠 Key Concept

The `except` block:
- Executes **only when an error occurs**
- Can target **specific exception types**
- Can capture error details for logging or messaging
----------------------------------------------------------------------
## ✅ Best Practices

✔ Catch **specific exceptions** whenever possible  
✔ Avoid bare `except` in production code  
✔ Keep exception messages clear and meaningful  
✔ Handle only what you can recover from

The `except` statement transforms Python programs from fragile scripts
