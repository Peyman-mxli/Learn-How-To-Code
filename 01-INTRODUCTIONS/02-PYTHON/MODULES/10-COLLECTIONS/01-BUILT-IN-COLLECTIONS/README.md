✨ Overview
Python provides several built-in collection data types used to store,
organize, and manage multiple values efficiently.
These collections are fundamental in professional Python development
and are used across real-world systems such as APIs, databases,
automation tools, and data processing pipelines.
This module introduces collections using their official classification:
Sequences, Sets, and Mappings.
=============================================================================
🎯 Objectives
By completing this module, I will understand Python’s built-in collections,

and write clean, scalable, and professional Python code.
=============================================================================
📦 Built-in Collection Categories
Python groups collections into three main categories:
Built-in Collections
│
├── Sequences
│ ├── list
│ ├── tuple
│ ├── str
│ └── range
│
├── Sets
│ ├── set
│ └── frozenset
│
└── Mappings
└── dict
Each category is designed to solve specific data modeling problems.
============================================================================
🔹 Sequences
Sequences store ordered collections of items where each element
has a defined position (index).
They are used when order matters, when accessing data by position,
and when iterating through elements.
============================================================================
🔹 Sets
Sets store unordered collections of unique elements.
They are ideal when uniqueness is required, when performing
membership checks, and when using mathematical set operations.
============================================================================
🔹 Mappings
Mappings store key–value pairs.
They are used when data needs labels, fast lookup is required,
and when modeling structured real-world data such as users,
settings, or records.
