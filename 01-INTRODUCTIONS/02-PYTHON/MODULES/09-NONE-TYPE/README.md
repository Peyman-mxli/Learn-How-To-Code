🧵 Module — NoneType— (None)
---------------------------------------------------------------
✨ Overview
----------------------------------------
In Python, None represents the absence of a value. It is not zero,
not an empty string, and not False. Instead,
None is a unique object of type NoneType.
Understanding None is essential for:
writing correct conditional logic
designing functions with meaningful return values
avoiding common bugs
reading and writing professional Python code
This module focuses on how None works, how it behaves 
in expressions, and how it should be used properly.
---------------------------------------------------------------
🎯 What I will learn in this module
By the end of this module, I will be able to:
Understand what None represents in Python
Identify the NoneType object
Use None correctly in conditions
Compare None safely using is and is not
Understand default return values of functions
Avoid common beginner mistakes related to None
Use None intentionally in real programs
----------------------------------------------------------------
🧠 Why NoneType matters
Many bugs in Python programs come from misunderstanding None.
Common situations where None appears:
a function that does not explicitly return a value
a variable that is intentionally uninitialized
optional parameters with default values
failed searches or lookups
Knowing how to handle None properly helps you:
write safer code
prevent runtime errors
design clearer APIs
improve code readability
