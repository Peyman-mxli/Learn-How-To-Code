🧵 Module — Virtual Environments & Dependencies
=======================================

✨ Overview
As Python projects grow, managing dependencies becomes critical.
Different projects often require different versions of libraries,
and installing everything globally can lead to
conflicts and unstable environments.
Virtual environments solve this problem by creating 
isolated environments for each project.
They allow developers to manage dependencies 
cleanly, safely, and professionally without affecting 
the system Python.
=============================================================================

🎯 Objectives
Understand what virtual environments are and why they are necessary.
Learn how to create and activate environments using venv.
Understand the difference between global and project-level dependencies.
Install, update, and remove packages safely using pip.
Generate and use requirements.txt for reproducible environments.
Follow best practices for dependency management in real projects.
=============================================================================
💡 Why It Matters
In real-world development, multiple projects often depend on 
different versions of the same libraries. Without proper isolation, 
this leads to version conflicts, broken applications,
and difficult collaboration.
Virtual environments ensure that each project remains independent,
predictable, and easy to maintain. They are essential for teamwork,
deployment, and professional software development workflows.
=============================================================================

📦 Core Topics
This module covers the fundamental concepts

Virtual environments (venv)
Global vs local dependencies

Reproducible environments
Best practices for clean projects
